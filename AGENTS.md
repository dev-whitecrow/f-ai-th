# AGENTS.md for Overall Project
0. **MENDATORY** 해당 repo의 모든 파일과 내용은 사용자의 직접적인 수정 명령이 있는 경우만 수정한다.
1. 너와 사용자의 Communication deviation을 최소화하고 정확히 요구사항을 수행하기 위해서, 첫 프롬프트에 대한 대답으로 Context에 필요한 요구사항들을 질문하는 습관을 가질것.
> <task> 또는 <instructions>: 수행해야 할 핵심 지시사항과 규칙
> <context> 또는 <document>: 작업에 필요한 배경 정보나 분석할 원본 데이터
> <examples>: 원하는 결과물에 대한 구체적인 예시 (지시사항과 혼동을 막기 위해 분리)
> <thinking>: 복잡한 문제나 분석이 필요할 때, 최종 답변을 내놓기 전 단계적 추론 과정을 거치도록 유도
등으로 필요한 부분을 명확하게 template을 만들어서 질문하도록.

2. 문제가 처음 생각했던 방식으로 해결이 되지 않는 경우 더 진행하지 말고 반드시 사용자에게 물어볼것. 또한 모르는 개념을 입력 받거나 사용해야하는 상황이 생기면 반드시 웹검색 tool call로 검색한 뒤, 그 결과물을 토대로 판단할것. 

3. 해당 세부 폴더의 AGENTS.md를 참조할 것 (이 파일의 최하단에 색인이 있음)

4. skill을 만들때 순서
- skills.sh의 npx skills 사용 (단 좋아요가 1000개 이상인 검증된 것만; 악성코드, 탈취 방지 목적)
- 직접 작성해야하는 경우 [새로운 스킬을 짜는 가이드](https://blakecrosley.com/ko/blog/agents-md-patterns)의 가이드라인 및 주의사항을 꼼꼼히 확인하여 작성
- 예시를 작성하지 않은 개조식의 짧고 명료하게 작성
- 강제사항, 유의사항 등은 LLM이 지나치기 어렵도록 영어로 표기(**MENDATORY**, **CAUTION**, **IMPORTANT** 등) 

5. 모든 생성 텍스트 전반 가이드라인
- SOLID 원칙으로 최소한의 필요한 내용만 담는다. (특히 llm 자체에 학습돼있는 일반적인 내용은 넣지 않는다. ex. "SOLID의 정의, Lean 방법론이란?")

- 결과물 내 불필요한 서두 텍스트는 모두 생략.
- 명확한 context 가 필요한 경우 xml로 태깅하여 제공할것. (어떤 것이 무슨 context 인지 LLM이 알 수 있도록)
  > **권장 태그**: `<context>`(배경정보), `<instructions>`(지시사항), `<constraints>`(제약조건), `<examples>`(예시), `<output_format>`(출력형식)
  > 일관된 태그를 사용하고, 복잡한 지시는 태그를 중첩하여 계층 구조를 명확히 할 것.
- 모든 파일에는 YAML Front Matter를 포함시켜 metadata 정의
> AGENTS, skill, 다른 문맥이 해당 파일을 읽었을 때, 한 눈에 문서 내용을 파악, 가늠할 수 있는 내용을 간략하게 작성.

6. Context-as-Code Rules
- git worktree 기능을 적극적으로 
- Error, 사용자가 지적했던 부분, 쉽지 않았던 이유와 lesson learn등을 PR 에 작성 (실패원인 등은 다른 곳에는 남길 필요 없음 - 오히려 llm context contamination을 유발)

7. py 파일 등을 생성하여 작업해야하는 경우 88_Temp 에 넣어두어 사용자가 언제든 지울 수 있는 휴지통처럼 사용한다.

8. 프롬프트 작성 프레임워크 (사용자의 프롬프트를 받아 최종 prompt로 정제)
- **구체적 지시**: 뛰어난 신입사원에게 지시하듯 정확하고 구체적인 요구사항 작성.
- **예시 제공 (Multishot)**: 추상적인 설명이라 판단하는 경우 1~3개의 구체적인 예시를 요청. 이를 사용자에게 받아 요약하여 프롬프트에 append
- **출력 형식 지정**: 표, 단어 수, 포맷 등 결과물의 형태를 명확히 지정.
- **영어로 작성** **MANDATORY**: 프롬프트는 영어로 작성하되 결과물은 한국어로 출력하도록 (프롬프트 마지막 부분에) 명령. (모든 과정은 영문 입력 출력이 훨씬 토큰 효율적이고, 정확함)

9. 단계적 추론 유도 (Chain of Thought)
- 복잡한 분석이나 코딩 작업 시, 최종 결과를 내기 전에 먼저 `<analysis>` 등의 태그 안에서 llm이 이 문제 해결 과정을 단계적으로 생각(Think step by step)하도록 유도할 것.
